# T-GEO 核心方法论与竞争壁垒

> 这份文档回答三个问题：**AI 到底凭什么推荐一家本地店？我们怎么撬动它？怎么做到别人抄不走的行业领先？**
> 不讲框架口号，只讲机理、杠杆和可交付的结果。配合 `GEO_PRD_本地生活版.md` 一起看。

---

## 第一部分：先认清真相（不认清，所有策略都是猜）

### 1.1 AI 推荐本地店，靠的是三个「面」，不是一个算法

用户问「推荐一家XX」，大模型的答案来自三处叠加：

1. **训练语料面**（模型脑子里记住的）——来自截止训练时全网公开语料。知名老店占便宜，新店/腰部店几乎"不存在"。
2. **实时检索面**（模型现查的）——豆包、元宝、通义千问、Kimi探索版、文心一言、ChatGPT、Perplexity 等会现场检索网页/平台再作答。**这是腰部商家唯一能短期撬动的面。**
3. **平台自有索引面**——抖音AI搜、小红书AI搜调用的是它们自己的 UGC 库；地图类(高德/苹果地图)、点评类(大众点评/美团)是结构化事实来源。

> **结论**：GEO 不是"优化一个网站"，是**让你的店在这三个面里同时"存在、一致、且被权威佐证"**。

### 1.2 三条颠覆认知的硬数据（来自第三方研究）

- **AI 本地可见度比 Google 难 3–30 倍**：第三方测评显示，ChatGPT 仅推荐约 **1.2%** 的本地点位、Gemini 约 **11%**、Perplexity 约 **7.4%**，而 Google 本地"三连包"出现率约 **35.9%**。AI 极度挑剔——这对我们是坏消息也是好消息：**门槛高 = 做进去就是稀缺资产。**
- **没有付费位**：ChatGPT / Perplexity / Claude 的回答里**买不到位置**，不能竞价。这意味着竞争对手有钱也插不了队，**只能靠 GEO 工程赢**——这正是付费意愿的来源。
- **约 86% 的引用来自"商家自己能控制的信源"**（官网、平台收录、点评）。还有研究显示被 AI 推荐的店**平均评分更高**（ChatGPT 侧约 4.3 星），且 **NAP（名称/地址/电话/营业时间/品类）一致性**显著提升被推荐概率。酒店侧更夸张：全球酒店**只有约 1/6 对 AI 可见**，有结构化 schema 标注的不到 **11%**，而 AI 可见的酒店转化率约 **11.4%、是自然搜索的 2 倍多**。

> ⚠️ **数据口径说明（重要）**：上面这些百分比来自**海外英文研究**（ChatGPT / Gemini / Perplexity / Yelp / Foursquare / 海外酒店样本），**只能当方向参考，不能当国内事实直接引用**。国内主战场是 DeepSeek / 豆包 / Kimi / 元宝 / 通义千问 / 文心一言 / 智谱清言 等助手（全渠道清单见 `src/lib/platforms.ts`）+ 大众点评 / 美团 / 高德 / 百度地图 / 小红书 / 抖音 这些信源。结论方向是一致的（AI 极挑剔、买不进、可控信源占大头），但给客户讲数字时要么自己实测国内口径，要么标注"海外数据，仅供参考"。

> **这三条决定了我们的打法**：① 主攻"实时检索面 + 平台索引面"（短期可撬动）；② 死磕信源一致性与权威度（占 86%、且可控）；③ 把"做进 AI 推荐"当成稀缺资产卖（难、不可买、可量化）。

## 第二部分：由机理倒推的「可撬动杠杆」（按 ROI 排序）

不是所有事都值得做。按"撬动力 ÷ 成本"排序，资源永远先砸前三个：

| 优先级 | 杠杆 | 为什么有效 | 具体动作 |
|---|---|---|---|
| ★★★★★ | **信源一致性矩阵** | 占 86% 引用、可控、AI 靠它建立"置信" | 让 NAP+品类+卖点在 大众点评/美团/高德/官网/百科 **完全一致**，消灭过时与矛盾信息 |
| ★★★★★ | **结构化实体知识** | AI 靠"实体-属性"匹配意图；多数店是非结构化文字，AI 读不准 | 把店做成知识图谱：招牌项目、场景标签、资质、价位等都结构化，并用 Schema.org 标注官网 |
| ★★★★☆ | **高权重信源投喂** | 实时检索面会优先引用权威平台 | 在 知乎/百科/垂直媒体/点评 铺 E-E-A-T 内容，建立"被引用源" |
| ★★★★☆ | **意图覆盖广度** | AI 是"提及率"不是"排名"，要在尽可能多的意图问句里出现 | 用行业意图矩阵(100+)系统化覆盖长尾场景，而非只盯一两个词 |
| ★★★☆☆ | **评分/口碑信号** | 高分店被推荐概率显著更高 | 联动点评运营提升真实评分、引导真实好评(不刷量) |
| ★★★☆☆ | **平台 UGC（小红书/抖音）** | 喂养平台自有 AI 索引 | 人机协作：出内容包+真实达人发布(不养号) |

> **反面清单（别浪费钱）**：纯做百度 SEO、买僵尸评论、群发软文、追求"实时 100% 全平台覆盖"。这些要么过时、要么违规、要么伪需求。

## 第三部分：T-GEO 核心运行工作流（这是产品的"发动机"）

一个客户进来，跑的是这条**可量化、可复制、越跑越值钱**的闭环：

```
①诊断基线 → ②定位差距 → ③结构化资产 → ④信源工程 → ⑤内容投喂 → ⑥再监测 → ⑦归因复盘 → 回到④
        （量化现状）   （找撬动点）   （建知识图谱）  （补一致性+权威） （人机协作分发） （证明进步）  （沉淀因果）
```

### 逐环节做什么 / 产出什么

**① 诊断基线（量化"AI 现在怎么看你"）**
- 跑 100+ 行业意图 × 10 平台 × 3 次，得出基线 **T-LBR** 与 **TopRec@3**。
- 抓四件事：是否被提及 / 排第几 / 描述是否准确 / 被谁压制。
- 产出：一份"老板看得懂"的现状报告（含截图证据），1 页摘要 + 明细。

**② 定位差距（找最该撬的点）**
- 对照行业基准 + 前 5 名竞品，标红"该被推却没被推"的意图。
- 输出"撬动优先级清单"：哪些是信源不一致、哪些是知识缺失、哪些是权威度不足。

**③ 结构化资产（把店变成 AI 读得懂的实体）**
- 把菜单/资质/媒体报道等非结构化资料 → 知识图谱节点（招牌项目、场景、资质、价位…）。
- 官网/落地页加 **Schema.org** 结构化标注（这一步多数同行不做，是低成本高回报洼地）。

**④ 信源工程（消灭矛盾 + 补权威）**
- 全平台 NAP/品类/卖点对齐到**完全一致**（这是 86% 引用的根）。
- 在高权重源（点评/百科/知乎/垂媒）建立"被引用资产"，按平台权重排优先级。

**⑤ 内容投喂（人机协作，不碰红线）**
- 基于知识图谱 + Prompt 库批量产 E-E-A-T 内容（场景化、客观、合规）。
- 出"内容包 + 分发清单 + 达人话术"，由客户/真实达人发布——**绝不自动养号/刷量**。

**⑥ 再监测（同口径复测）**
- 同样的意图矩阵周期性复跑，得出 T-LBR / TopRec@3 的**变化曲线**。

**⑦ 归因复盘（把"运气"变成"方法"）**
- 记录"做了什么动作 → 哪个平台哪个模型的答案变了"。
- 这条因果记录**进库沉淀**——这就是护城河（见第四部分）。

> 这套闭环对客户的意义：**每个月都能拿到一张"进步曲线 + 因果说明"**，续费理由是看得见的数字，不是玄学。

## 第四部分：怎么做到"行业领先"——5 个别人抄不走的壁垒

工具人人能做（调几个 API 而已）。真正的壁垒是下面这些**越用越厚**的东西，按重要性排序：

### 壁垒 1：垂直「意图×知识 schema」深度（最现实的起点）
- 通用工具给你泛泛的关键词；我们有**餐饮/SPA/酒旅各自 100+ 意图矩阵 + 专属知识图谱字段**，知道"每个意图下 AI 到底看重哪几个属性"。
- 例：SPA"正规不踩雷"意图，AI 看重的是资质/卫生/技师年限/真实口碑——通用工具根本不知道要喂这些。
- **怎么建**：每服务一家店，就把该行业意图-属性映射打磨深一层。**这是"在大象腿边长草"的草根优势**，迈富时这种横向平台做不深。

### 壁垒 2：因果级「推荐机制」数据资产（真护城河）
- 跑得越多，越能积累一张经验地图：**"在豆包上做 X 信源 → 让 '正规SPA' 意图排名上升 Y"** 这种**因果对**。
- 这是别人**砸钱也买不到**的私有数据，是从"猜"到"知道"的分水岭。
- **怎么建**：第③~⑦环节的每次"动作→结果"都结构化入库（`项目交接说明.md` 的 `mention_records`/归因表）。半年后你比任何人都懂"本地生活的 AI 推荐机制"。

### 壁垒 3：敏感行业的「信任工程」能力（最难、最值钱）
- SPA/按摩行业关键词容易被模型安全过滤、被降权——**能把敏感品类安全地做进 AI 推荐，是稀缺硬本事**。
- 打法：统一用"养生/推拿/中医调理/身心放松"替代词建知识图谱；强化资质/卫生/技师认证等信任信号；关联中医协会/康复医学等权威。
- 谁能稳定攻克这个，谁就拿下这个高客单、高复购、同行最焦虑的细分。

### 壁垒 4：「人机协作分发 + 真实达人/信源网络」（最难自动化的最后一公里）
- 内容能 AI 生成，但"送到 AI 会看的地方"靠的是矩阵账号 + 真实达人 + 合规分发——这是连迈富时都没真正解决的灰色地带。
- 谁先建起三行业的达人/MCN 池 + 合规分发 SOP，谁就有结构性优势。**这是组织能力壁垒，不是技术壁垒。**

### 壁垒 5：可量化的「效果承诺」与口碑案例
- 因为 AI 推荐**不可买、可量化**，我们能做别人不敢做的承诺：**"90 天 TopRec@3 提升不达标退 50%"**。
- 每个达标案例 → 脱敏成行业案例 → 反哺销售信任。**案例库本身就是复利资产。**

> **一句话总结壁垒**：技术只是入场券；真正领先来自 **① 垂直 schema 深度 + ② 因果数据资产 + ③ 敏感行业信任工程 + ④ 真实分发网络 + ⑤ 可证明的效果**。前两个靠产品天天沉淀，后三个靠运营死磕。

## 第五部分：三行业差异化打法（具体到杠杆）

### 餐饮
- **高价值意图**：城市+菜系推荐、商务宴请/家庭聚餐/约会、招牌菜、深夜营业、外地朋友本地菜。
- **关键杠杆**：招牌菜结构化 + 大众点评/小红书信源一致 + "场景化"种草（达人代发）。
- **赢的标志**：在"{城市}{菜系}推荐""适合商务宴请"这类意图里进 Top 3。

### 按摩 SPA（最难也最赚）
- **高价值意图**：正规/不踩雷、症状调理(颈椎/腰椎/失眠)、情侣SPA、孕妇可做、出差住店附近。
- **关键杠杆**：**信任工程**（资质/卫生/技师年限）+ 替代词知识图谱 + 健康科普权威源（知乎健康/丁香等）。
- **赢的标志**："正规/不踩雷/推荐"类意图 Top 3 命中——这是该行业的命门。

### 酒旅
- **高价值意图**：X天行程含住、景点附近、亲子/情侣/商旅、看海景/雪山、避坑住宿。
- **关键杠杆**：**Schema.org 结构化标注**(酒店侧极度稀缺、洼地)+ 进入"行程类"内容(马蜂窝/小红书/知乎攻略) + OTA 之外的非标引用。
- **赢的标志**：出现在"{城市}3天行程推荐"这类**行程类**答案的住宿环节里。

## 第六部分：怎么向客户证明"真有用"（实际帮到用户）

空泛的报告留不住客户。每个客户必须拿到**可量化、可对比、可归因**的三样东西：

1. **北极星指标 TopRec@3**：核心意图被 AI 推为前 3 的比例。签约 90 天内提升 **≥25 个百分点**为合格（行业基准：餐饮≥40%、SPA≥30%、酒旅≥35%）。
2. **T-LBR 进步曲线**：总分 + 4 维度(曝光/定位/信任/转化倾向)的周/月变化，与竞品同框对比。
3. **因果说明**："本月做了 A/B/C → 在豆包/Kimi 的 X 意图里从未提及变成 Top 2"，附答案截图证据。

> **交付节奏**：诊断报告(进店)→ 月度内容包 → 月度进步报告 → 季度复盘会。让客户**每月都看见数字在涨**。

## 第八部分：数据资产飞轮（护城河怎么复利）

```
服务一家店 → 沉淀「意图-属性映射」更深 + 「动作-结果」因果对更多
        ↓
诊断更准、撬动更狠、达标更快、成本更低
        ↓
更敢做效果承诺 → 更多案例 → 更多客户 → 沉淀更多数据
        ↑__________________________________________↓
                （越跑越懂"AI 推荐机制"，越难被超越）
```

> 这就是为什么要**死守三个行业、把闭环数据全部入库**：你卖的不是工具，是一台**越用越聪明的"本地生活 AI 推荐机制"引擎**。这台引擎，就是核心竞争力。

## 第九部分：竞品深度对标——迈富时 T-GEO™

> 这是头号对标对象。先看清它是谁、强在哪、够不着哪，才知道我们的楔子插在哪里。

### 9.1 它到底是什么

- **身份**：珍岛集团旗下，上市营销 SaaS 老牌（10 年+ 营销服务积累），自研 **AI-Tforce 营销大模型**（宣称千亿级参数）。
- **产品**：**T-GEO™ 生成引擎认知工程模型（GEC）**，主打"行业唯一把 GEO 从碎片化操作升级为系统化工程"的话语权。
- **五层认知架构**：
  1. 用户 AI Query 行为层（解析用户怎么问）
  2. 语义空间建模层（缩短品牌与问题的语义距离）
  3. 生成引擎认知机制层（顺应模型偏好）
  4. 品牌语料训练与信源控制层（把品牌"训练进"AI 信赖的内容体系）
  5. 生成反馈与强化学习层（让推荐持续稳定）
- **服务模式**："培训 + 工具 + 资源"生态 + 专属客户成功经理 + 7×24 响应。

### 9.2 它宣称的数据——要打折看

意图预测 >85%、语义匹配 **99.92%**、说服力 +40%、引用率 >80%、效果达成率 **99%**、**1:6 ROI**……
> **关键判断**：这些数字**绝大多数出自自媒体软文 / PR 通稿（自报、未经独立验证）**。99.92%、99% 达成率、1:6 ROI 都是营销话术，不是审计结论。**别被它的数字吓住，也别在自己材料里抄它的数字。** 它真正强的是"工程化叙事"和 PR 势能，不一定是效果本身。

### 9.3 它的真实护城河（不是技术）

①上市公司资源 + 十年营销服务老客户基本盘；②**B2B 销售铁军**；③自研营销大模型 + RL 闭环（强品牌叙事）；④工程化、标准化的交付能力；⑤"行业唯一系统化"的话语权/PR。

### 9.4 它够不着、瞧不上的地方（=我们的楔子）

| 维度 | 迈富时 | 我们的机会 |
|---|---|---|
| 行业 | 横向全行业，**案例集中在 B2B 工业制造**（化工/建材/制造/包装/塑胶/文具/环保/自动化） | 垂直**本地生活三行业**，做深意图/信任/场景 |
| 信源战场 | 重官网 / B2B 媒体 / 企业语料 | 本地推荐的"地面真相"在**地图 / 点评 / 小红书 / 抖音 UGC**——它不在这深耕 |
| 客单 | 数十万–百万，盯央国企/上市公司 | 单店数千–数万，它的销售模型**覆盖不了、也不想覆盖** |
| 赢的关键 | "更大的模型 + RL"叙事 | 本地靠**信源一致性 + 真实达人分发 + 敏感行业信任工程**，不是更大的模型 |

> **战略结论（与 PRD §2.2 一致）**：**不要正面竞争。** 做它够不着的"本地生活单店脏活累活"。我们**不抄它的模型/RL**，但**抄它的"工程化 + 标准化 + 强客户成功 + 可量化交付"这套服务方法论**，落进本地生活垂直里——它的方法论是对的，只是没下沉到我们这块地。

## 第十部分：本方法论的勘误与补强（自我审查）

> 对前八部分做一次诚实复盘，避免把文档当圣经用。

### 勘误
1. **数据口径**（已在 1.2 加说明）：硬数据来自海外英文研究，仅作方向参考；国内主战场平台不同，数字不可当国内事实引用。
2. **三个"面"的可撬动性**：训练语料面短期几乎撬不动（要长周期沉淀），别给客户承诺"很快进模型脑子"；短期成效主要来自**实时检索面 + 平台索引面**，要讲清楚。

### 重要补强（前文漏掉、但很关键）
3. **把"地图 + 点评"独立为第一可控支柱**：海外是 Foursquare 喂 ChatGPT、Yelp 被引约 33%；国内对应**高德/苹果地图 + 大众点评/美团**。本地生活的"地面真相"主要来自这里，优先级**等同甚至高于官网**。第二部分的杠杆表应把它单列为最高优先级。
4. **"AI 错配标签 / 失实过时信息纠错"是见效最快的杠杆**：把高端店写成快餐、被已倒闭同行压制、营业时间/价位过时——这些一改立竿见影，最能在第一个月就打动客户。应在工作流第②③环节**前置**。
5. **Agent / MCP 范式迁移对冲**（前瞻、也是相对迈富时的优势）：未来 AI 推荐可能从"语义匹配"转向**Agent 直接调用美团/高德/携程的结构化接口**。对冲 = 把结构化数据做干净 + 把知识库做成 **MCP / Agent 可调用**形态。迈富时是 B2B 企业语料导向，这块反应会比我们慢。
6. **测量必须严谨**：LLM 回答有**随机性**，还受**地域 / 账号个性化 / 模型版本漂移**影响。"提及率"是**分布不是单点**——必须固定口径、多次采样、记录模型版本与查询地域，否则"进步曲线"不可信，承诺也无法兑现。
7. **ROI 归因要诚实**：AI → 到店**很难直接追踪**。用代理指标：点评/美团到店增量、落地页 UTM、到店时"您怎么知道我们的"问卷。"90 天 TopRec@3 不达标退 50%"这类承诺**必须绑定可复测的口径**，否则是空头支票。
8. **明确不与迈富时拼模型/RL**（写进壁垒章的前提）：我们**不自研营销大模型、不硬做强化学习**（PRD §3.1 也说不要硬做 RL）。核心竞争力 = 垂直 schema 深度 + 本地信源事实层 + 真实分发网络 + 敏感行业信任工程 + 可量化交付。这是"轻团队跑得比大象快"的根本。
