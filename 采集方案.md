# 原料采集方案（当前推荐 · 会演进）

> ⚠️ **这是一份方案，不是定论。** 现成工具和平台政策变化很快，出现更优解时直接替换本表即可。
> 工程上：采集方式在诊断引擎里只是渠道适配器的一个**可换字段**（method），换方案 = 换适配器，主流程零改动。
> 最近更新：2026-05。配合 `platforms.ts`（渠道清单）、`intents.ts`（意图矩阵）一起看。

---

## 0. 两条铁律（不随方案变）

1. **能用官方 API 就用 API**（占大头、零封控、稳定可规模化）。
2. **采集只负责拿到「AI 答案全文」**；5 个评分信号（提及 / 排名 / 描述准确度 / **信源权威度** / 情感）都是在**答案文本上解析**得到的，不是额外的采集动作。
   - 例：「信源权威度」= 从答案里识别它引用了哪些来源（大众点评/百科/小红书…）再查权重，**不需要单独再爬**。

## 1. 渠道 → 采集方式映射（当前方案，可替换）

| 渠道 | 采集方式（method） | 现成可套项目/路线 | 备注 |
|---|---|---|---|
| DeepSeek / 通义千问 / 文心一言 / 智谱 / Kimi / 讯飞星火 / 混元 | `api` | 各家官方开放平台 | 商用首选，零封控 |
| 豆包 | `api` | 火山方舟 | |
| ChatGPT / Gemini / Claude / Perplexity | `api` | 各家官方 API | |
| **小红书搜** | `mediacrawler` | **MediaCrawler**；要更稳可加 **Spider_XHS**（逆向签名、走 HTTP 接口） | 社媒平台爬虫的事实标准 |
| **抖音搜** | `mediacrawler` | **MediaCrawler**（抖音搜索+视频+评论） | |
| **元宝**（腾讯） | `token_replay` / `agent` | token 重放（参考 LLM-Red-Team free-api 打法）；底层混元有腾讯云 API 作近似 | 无成熟专用开源，自接或外包 |
| **夸克 AI**（阿里） | `token_replay` / `agent` | 同上；底层通义有官方 API 作近似 | |
| 百度AI搜 / 微信AI搜 | `manual` / 外包 | 较封闭，无成熟开源 | 人工抽样兜底 |

> 「能否用 MediaCrawler」一句话版：**小红书搜、抖音搜 → 用；元宝、夸克 → 不用**（它只支持社媒平台，不支持 AI 助手）。

## 2. 现成项目参考（套用别人成功案例）

- **MediaCrawler**（NanmiCoder，~28k★）：小红书/抖音/快手/B站/微博/知乎/贴吧 内容+评论+搜索。免逆向、登录态缓存、代理池。开源版 **非商用**，商用走商业版 **MediaCrawlerPro**（需联系作者询价）或外包。
- **Spider_XHS**（cv-cat）：逆向小红书签名、封装核心 HTTP 接口，比浏览器爬更快更稳。
- **LLM-Red-Team `*-free-api`**：把 C 端聊天 AI 逆向成 OpenAI 接口的 **token 重放**打法范例。**标注仅供测试，商用请走官方平台**——我们只借鉴其「一次登录拿凭证、复用内部接口」的原理，用自有账号合规实现。
- **browser-use / Skyvern / Stagehand**：AI Agent 操作浏览器、读页面答案，抗 UI 改版，适合元宝/夸克这类没有现成爬虫的入口。

## 3. 工程落地（让方案可替换）

诊断引擎用**渠道适配器模式**，`method` 来自上表、是可换字段：

```ts
// worker/channels/types.ts
export type CollectMethod = 'api' | 'mediacrawler' | 'spider_xhs' | 'token_replay' | 'agent' | 'manual';
export interface ChannelAdapter {
  id: string;            // 对应 platforms.ts 的 id
  method: CollectMethod; // 当前用哪种采集方式（换方案只改这里 + 对应适配器）
  query(intent: string): Promise<{ answer: string; citations?: string[] }>;
}
```

主流程只调 `adapter.query(intent)` 拿答案 → 统一解析 5 信号 → 算 T-LBR。
**换采集方案 = 换某个渠道的适配器实现，主流程和评分逻辑都不动。**

每条采集结果建议记：`source_method` / `confidence` / `model_version` / `captured_at`，分数按可信度加权，报告标注真实覆盖率（不假装 100%）。

---

## 4. 合规红线（不随方案变）

- 只采**公开的 AI 答案/搜索结果**做监测；不登录他人账号、不绕付费墙、不抓个人信息、遵守各平台条款与 robots。
- 高风险入口优先走**持牌厂商/商业版**或**人工抽样**，把合规风险外移。
- 遵守《数据安全法》《个人信息保护法》；控制频率、不干扰平台正常运行。

---

> 一句话：**API 打底，小红书搜/抖音搜 用 MediaCrawler，元宝/夸克 用 token 重放或 Agent**——这是当前方案；method 字段化，随时可换更优解。
